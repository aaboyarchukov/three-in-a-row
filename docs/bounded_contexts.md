# Границы системы

## Части системы

Должны быть:
1. Генерация компонентов на доске
2. Подсчет комбинаций при перестановке элементов
3. Управление игрой - отслеживание ее конца + начало новой игры
4. Управление игроком и его действиями
5. Управление компонентами на доске
6. Подсчет статистики игрока

## Главные подсистемы

1. Система, отвечающая за управлением состоянием доски (перемещение элементов, генерация новых, подсчет комбинаций на доске)
2. Система, отвечающая за подсчет статистики игрока - ходы и очки
3. Система, которая управляет игрой - ее начало, окончание по определенному маркеру (например, определенное количество очков или ограничение по времени)
4. Бонусная система - выдает бонус игроку, управляет использованием бонусов

## Метафоры

Из метафор мы строим свой UL:
1. Доска - игровое поле, на котором расположены игровые элементы (компоненты)
2. Компоненты - объект на поле, из которых можно создавать комбинации
3. Игра - сессия, во время которой Игрок меняет состояние Доски и зарабатывает Очки
4. Игрок - пользователь Игры
5. Бонус - возможность удалять Компоненты с Доски различными способами (зависит от определенного бонуса)
6. Очки - баллы игрока за собранные Комбинации Компонентов
7. Ходы - один шаг игрока - когда игрок меняет местами элементы
8. Комбинация - несколько подряд идущих Компонентов
9. Статистика - набор значений по определенным категориям: Очки, Ходы


## Функциональность

1. Игрок:
	1. Начала и окончание Игры
	2. Переставление Компонентов на Доске
	3. Использование Бонусов
2. Игра
	1. Хранение Статистики
	2. Подсчет Статистики
	3. Подсчет Комбинаций на Доске
	4. Контроль окончания Игры
	5. Выдача бонусов

2. Доска
