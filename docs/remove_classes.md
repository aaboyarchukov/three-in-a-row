# Отбраковка плохих классов

## Критерии отбраковки

Нам необходимо отбраковать также плохие классы, которые будут только мешать проектированию и ломать иеррахию.

Отбраковать классы можно по следующим критериям:
- **класс не представляет из себя структуру данных с определенным набором формализованных действий** (если например класс просто является инкапсуляцией какой-то функции, то он не подходит); также класс можно отбраковать по названию, например, он назван императивным глаголом или инфинитивом **(должно быть существительное или глагол, описывающий свойство с окончанием -able)**
- **класс отвечает на вопрос "этот класс делает ..."**, класс может выполнять несколько действий, которые направлены на выполнение единой ответственности, но класс не должен делать что-то одно - например "класс записывает в БД пользователей" - именно одного действия быть не должно, одна ответственность - да
- **класс начинает или продолжает иерархию еще до окончания этапа анализа** (зачастую наследование не должно уходить больше чем на 5 уровней, с учетом General и Any), то есть мы не должны начинать строить иерархию до того, как закончим этап анализа и не определим основные классы -> **спешить с классификацией сущностей никогда не стоит**
- **класс без методов** (возможно не определили для этого АТД набор методов)
- **класс без полей** (или с небольшим их количеством) и наследует только поля родителя
- **класс не содержит команды, а только запросы** (исключения: инкапсулирование неизменяемого класса (коннектор к БД, сетевой драйвер), льготное наследование, паттерны проектирования)
- **в классе смешаны две абстрактные сущности** - ошибка проектирования, когда пытаются искусственно соединить две сущности

## Кандидаты на отбраковку

- **Cell** - в задумке данный класс нужен для того, чтобы устанавливать Компоненты, проверять на пустоту, как будто по критериям он просто инкапсулирует сам класс Компонент. Получается, что класс **Cell** можно спокойно убрать
- **BoardSets** - лишняя абстракция, которая никак не ускорит алгоритм подсчета последовательностей, можно навешать например `numpy`, для повторного использования, который будет работать с нашей матрицей
