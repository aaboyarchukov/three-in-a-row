# Классы проектирования

**BonusFabric** - (Паттерн Фабрика - позволяет использовать один интерйфейс у разных видов бонусов - то есть необходимо создавать объекты, которые объеденены одним доменом/доменной сущностью - в данной случае - Бонусная система)

**BonusFacade или BonusBridge** - (Паттерны Фасад и Мост - для разделения классов и работой именно абстракцией, а не конкретной реализацией, нужно в виду нескольких видов бонусов)

**BoardSingltone и UserSingltone** - (Паттерн Одиночка - используем для доски и пользователя, чтобы пользоваться именно одним экземпляром, не плодя копии)

**BoardWatcher** -  (Паттерн Наблюдатель - производит наблюдение за состоянием доски для последующих действий - подсчет статистики, истории ходов)
