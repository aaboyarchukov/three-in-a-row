# Классы реализации

При анализе проекта явно напрашиваются классы по работе с компонентами на поле, их подсчету, генерации; для хранения и подсчета статистики; хранение истории ходов:

- **BoardSets** - массив сетов, в которых будет происходить поиск 3 в ряд
- **StatisticMap** - словарь со статистическими данными, которые необходимо подсчитывать, а также хранить
- **MoveQueue** - очередь совершенных ходов - как история игрока
- **BonusMapArrays** - ассоциация между конкретным бонусом и массивом из координат элементов, которые будут удалены (так как бонусы могут удалять несколько элементов в разных позициях, так будет эффективнее всего)
- **ComponentsQueue** - очередь в которую будет подаваться очередной сгенерированный компонент, который будет помещен на карту
